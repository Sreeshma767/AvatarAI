C. Parent POM â€” replace the pom.xml in MultiModule with the exact content below

Open MultiModule/pom.xml and replace its contents with:

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
                             https://maven.apache.org/xsd/maven-4.0.0.xsd">

  </properties>

F. Build order and commands (Eclipse + CLI)
Recommended: build everything from the parent (this ensures correct reactor order)

CLI (command line) â€” in a terminal navigate to the parent project folder (the folder that contains pom.xml for MultiModule) and run:

Maven will:

build MultiModuleChild1 first (since Child2 depends on it),

then build MultiModuleChild2,

In Eclipse

If you try to build MultiModuleChild2 alone before Child1 / parent are installed, the build will fail because MultiModuleChild1 artifact is not yet available.
